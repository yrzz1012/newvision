# 07 — 部署指南

## 前置条件
- GitHub 账号
- Vercel 账号（建议 GitHub 登录绑定）
- Supabase 账号
- 域名（可选）

---

## 1. Supabase 配置

### 创建项目
1. 登录 [supabase.com](https://supabase.com)
2. 点击 `New project`
3. 填写：Name=`3dspace`, Database Password（妥善保存）
4. Region 选择 `ap-southeast-1`（新加坡，国内访问快）
5. 等待初始化完成（约2分钟）

### 获取API Keys
进入 `Settings > API`：
- **Project URL**: `https://xxxxx.supabase.co`
- **anon public key**: `eyJ...` (公开，可暴露)
- **service_role key**: `eyJ...` (密钥，禁止暴露)
- **JWT secret**: `...` (用于签名的密钥)

### 数据库迁移
在 Supabase Dashboard 的 `SQL Editor` 中，依次执行：
1. `docs/04-数据库设计.md` 中的建表SQL
2. RLS策略SQL
3. Storage Bucket 创建

### Auth 配置
- `Authentication > Providers` → 启用 Email
- 禁用 `Confirm email`（开发阶段，生产再开启）
- 站点URL设置为 Vercel 部署域名

---

## 2. Vercel 部署

### 关联仓库
```bash
# 在项目根目录
git remote add origin https://github.com/YOUR/3dspace.git
git push -u origin main
```

1. 登录 [vercel.com](https://vercel.com)
2. `Add New > Project` → 导入 GitHub 仓库
3. Framework 自动检测为 Next.js
4. 配置环境变量：

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
SPLAT_TRANSFORM_PATH=/usr/local/bin/splat-transform
```

5. 点击 `Deploy`

### 生产域名
- Vercel 自动分配：`3dspace.vercel.app`
- 可在 `Settings > Domains` 添加自定义域名

---

## 3. splat-transform 部署

由于 Vercel 运行环境为 serverless，需要使用 Edge Functions 或在 Supabase 侧处理：

### 方案A：Supabase Edge Function
在 Supabase 创建 Edge Function 处理 `.ply → .sog` 转换。

### 方案B：GitHub Actions 钩子
上传文件后触发 GitHub Action 进行转换。

### 方案C：V1阶段手动转换
开发阶段先用 `splat-transform` CLI 本地转换后手动上传 `.sog`。

**V1 推荐方案C**，简化开发流程。

| 变量 | 值 | 说明 |

| `NEXT_PUBLIC_SUPABASE_URL` | Supabase项目URL | 前端使用，可公开 |

| `SUPABASE_SERVICE_ROLE_KEY` | service_role key | 仅服务端使用 |
| `NEXT_PUBLIC_SITE_URL` | 生产URL | SEO/OG配置用 |
